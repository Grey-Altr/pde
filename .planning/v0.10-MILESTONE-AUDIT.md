## E2E Flows (3/3 complete)

### Flow 1: idle_prompt → suggestion file
`hooks.json` idle_prompt → `idle-suggestions.cjs` hook → event gate check → `generateSuggestions()` engine → `/tmp/pde-suggestions-{sessionId}.md` written

### Flow 2: suggestion file → tmux pane display
`/tmp/pde-suggestions-{sessionId}.md` → `pane-suggestions.sh` polling loop (sleep 3) → ANSI-formatted display in P6

### Flow 3: suggestion file → CLI stdout
`/pde:suggestions` command → `commands/suggestions.md` → `pde-tools.cjs suggestions` → reads `/tmp/pde-suggestions-{sessionId}.md` → stdout

## Integration Checker Notes

Two architectural decisions documented by the integration checker (informational, not defects):

1. **Two distinct zero-state messages** — `AWAITING_MSG` in the engine vs. the pane/CLI "Waiting for PDE..." message serve different conditions (file exists but no phase data vs. file doesn't exist). Architecturally correct.

2. **`phase_complete` maps to phaseType `review`** which has no catalog section — falls back to `default` section. Intentional per Phase 72 decision log: `review` category is reserved for dynamically-generated artifact review entries, not catalog-sourced content.
