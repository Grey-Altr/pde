## Decisions Made

- AC-N verification is additive to existing per-task `<acceptance_criteria>` checks — both must pass before a task is done. The existing code-level grep checks run first, then the behavioral AC-N "then" clause verification runs second.
- Boundaries check fires before task execution (not after) so the executor can stop before making any modifications to listed paths.
- Plan-level `<acceptance_criteria>` block must be placed BEFORE `<tasks>` in PLAN.md — this is enforced by the planner prompt rule and is a prerequisite for the sharding.cjs extractPlanAcBlock() function (Phase 48 Plan 03) to work correctly without matching per-task acceptance_criteria blocks.
