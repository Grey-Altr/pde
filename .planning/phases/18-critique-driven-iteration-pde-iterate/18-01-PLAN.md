Create workflows/iterate.md following the 7-step PDE skill workflow pattern exactly as used in workflows/critique.md. The file structure is:

**Top-level sections (in order):** `<purpose>`, `<required_reading>`, `<flags>`, `<process>` (containing Steps 1-7), Anti-Patterns, `<output>`.

**`<purpose>` section:**
Critique-driven iteration of wireframes. Reads the latest CRT-critique-v{N}.md, applies open Action List findings as targeted edits to wireframe HTML (producing versioned WFR-{screen}-v{N}.html copies — originals NEVER overwritten), produces ITR-changelog-v{N}.md mapping each finding to applied or deferred status, updates critique report checkboxes and Resolved Findings table, updates ux/DESIGN-STATE.md Open Critique Items statuses, and (on run 3+) surfaces a convergence checklist with handoff recommendation. What Works items from the critique report are a read-only constraint — findings conflicting with What Works elements are deferred with documented reason.

**`<required_reading>` section:**
```
@references/skill-style-guide.md
@references/mcp-integration.md
```

**`<flags>` section:**
Table with these flags:
| Flag | Type | Behavior |
|------|------|----------|
| `--dry-run` | Boolean | Runs Steps 1-3 only, shows planned iteration scope (open findings, affected screens, version numbers). No files written. |
| `--quick` | Boolean | Skip MCP enhancements (Sequential Thinking). Apply findings without conflict reasoning. |
| `--verbose` | Boolean | Show detailed progress, timing per step, finding-level application details. |
| `--no-mcp` | Boolean | Skip ALL MCP probes. Pure baseline mode. |
| `--no-sequential-thinking` | Boolean | Skip Sequential Thinking MCP only. |
| `--force` | Boolean | Skip confirmation when changelog already exists for this iteration version — proceed silently. |

**`<process>` section — Step 1/7: Initialize design directories**
Identical to critique.md Step 1:
```bash
INIT=$(node "${CLAUDE_PLUGIN_ROOT}/bin/pde-tools.cjs" design ensure-dirs)

```
Halt on error with: `Error: Failed to initialize design directories.\n  The design directory structure could not be created.\n  Check that .planning/ exists and is writable, then re-run /pde:iterate.`
On success: `Step 1/7: Design directories initialized.`

**Step 2/7: Check prerequisites and discover artifacts** — Six sub-sections:

2a. Find latest CRT-critique-v*.md — Glob for `.planning/design/review/CRT-critique-v*.md`, sort descending by v{N} suffix, Read the highest version. HARD DEPENDENCY — if absent, halt with:
```
Error: No critique report found.
  Run /pde:critique first to generate findings, then retry /pde:iterate.
```
Store content as CRITIQUE_REPORT. Store version number as CRITIQUE_VERSION.

2b. Parse Action List checkboxes — Find the `## Action List for /pde:iterate` section in CRITIQUE_REPORT. Parse each line matching `- [ ] {summary} -- {severity}/{effort}` as an open finding. Parse `- [x] {summary} -- {severity}/{effort}` as already resolved (skip). Store open findings as OPEN_FINDINGS array. If zero open findings: output `All findings already resolved. Nothing to iterate.` and halt gracefully (not an error).

2c. Parse What Works table — Find the `## What Works` section in CRITIQUE_REPORT (between `## Summary Scorecard` and `## Findings by Priority` in the live CRT file — NOT in templates/critique-report.md). Parse each table row: `| {Element} | {What's Working} | {Perspective} | {Keep It} |`. Store as WHAT_WORKS array. If section absent: warn `Warning: No What Works section found in critique report. Proceeding without preservation constraints.` and continue with empty WHAT_WORKS.

2d. Match findings to wireframes — For each open finding, extract the screen slug from the Location field (first segment before ` > `). Glob for `.planning/design/ux/wireframes/WFR-{screen-slug}*.html`. If no matching wireframe for a finding: halt with `Error: Finding references screen '{screen-slug}' but no wireframe found at .planning/design/ux/wireframes/WFR-{screen-slug}.html. Ensure wireframes exist before iterating.` Store unique screen slugs as AFFECTED_SCREENS.

2e. Discover iteration depth — Glob for `.planning/design/review/ITR-changelog-v*.md`. Parse v{N} suffix from each match, extract max N. Set ITR_VERSION = max(N) + 1 (or 1 if none exist). Set ITERATION_DEPTH = ITR_VERSION (the run being produced, 1-based).

2f. Per-screen version gate — For each screen slug in AFFECTED_SCREENS:
  - Glob for `.planning/design/ux/wireframes/WFR-{screen}-v*.html`
  - If any exist: SOURCE = path of highest version; NEW_VERSION = max(existing v* suffix integer) + 1
  - If none exist: SOURCE = `.planning/design/ux/wireframes/WFR-{screen}.html` (original — read-only); NEW_VERSION = 1
  - OUTPUT_PATH = `.planning/design/ux/wireframes/WFR-{screen}-v{NEW_VERSION}.html`
  - Store per-screen: {slug, source_path, output_path, new_version}

Display: `Step 2/7: {count} open findings across {screen_count} screen(s). Iteration v{ITR_VERSION} will be produced.`

If `--dry-run`: display planned scope (open findings count, affected screens, version numbers for each screen, ITR_VERSION) and HALT. Do not proceed to Step 3.

**Step 3/7: Probe MCP availability**
Check flags first (same pattern as critique.md Step 3):
- `--no-mcp` or `--quick`: set SEQUENTIAL_THINKING_AVAILABLE = false, skip all probes
- `--no-sequential-thinking`: set SEQUENTIAL_THINKING_AVAILABLE = false, skip probe

3a. Sequential Thinking MCP probe (if not skipped):
Attempt `mcp__sequential-thinking__think` with test prompt `"Analyze the following: test"`. 30s timeout. Retry once on failure. Set SEQUENTIAL_THINKING_AVAILABLE = true/false.
Log: `  -> Sequential Thinking MCP: {available|unavailable (continuing without)}`

Display: `Step 3/7: MCP probes complete. Sequential Thinking: {yes|no}.`

**Step 4/7: Apply findings to wireframes**
For each screen slug in AFFECTED_SCREENS:
  1. Read the SOURCE file (source_path from Step 2f) into WIREFRAME_HTML
  2. Sort findings for this screen by severity: critical first, then major, minor, nit
  3. For each open finding targeting this screen:
     a. **What Works check:** Compare finding Location against each WHAT_WORKS element's Element path. If finding Location is a prefix-match or exact-match of any What Works element: classify as DEFERRED with reason "conflicts with What Works item: {element}". Do NOT apply.
     b. **Conflict check:** If SEQUENTIAL_THINKING_AVAILABLE and two findings suggest contradictory changes to the same element: use `mcp__sequential-thinking__think` with prompt: `"Two critique findings conflict on the same element. Finding A: {summary_a}. Finding B: {summary_b}. Which should take priority given the screen's role? Recommend which to apply and which to defer."` Apply the recommended one, defer the other.
     c. **Apply change:** Based on the finding's Suggestion field:
        - Attribute/value change (e.g., "change font-size from var(--font-size-xs) to var(--font-size-sm)"): Use Edit tool with exact string match on the target element line in WIREFRAME_HTML
        - Structural addition (e.g., "add skip-to-main-content link as first child of body"): Assemble modified version of full HTML with the structural change applied
        - If a finding's effort is "significant" and suggestion requires structural redesign beyond a targeted edit: classify as DEFERRED with reason "requires structural redesign beyond iteration scope"
     d. Classify as APPLIED (with description of change made) or DEFERRED (with reason)
     e. Accumulate into CHANGELOG_ENTRIES

Display per-screen progress: `Step 4/7: Applying findings to {screen_slug} ({N} of {total_screens})... {applied_count} applied, {deferred_count} deferred.`

**Anti-Patterns section (after Step 7, before `</process>`):**
NEVER do any of the following:
- Overwrite original wireframes: `WFR-{screen}.html` must NEVER be modified by iterate. Write tool only produces new `WFR-{screen}-v{N}.html` files.
- Apply findings that conflict with What Works: The What Works table is a read-only constraint. If a finding's Suggestion would change a What Works element, defer the finding and document the conflict.
- Update designCoverage without reading coverage-check first: `manifest-set-top-level` replaces the ENTIRE designCoverage object; skipping coverage-check resets flags set by other skills.
- Mark findings resolved without writing the wireframe: Update critique checkboxes AFTER the versioned wireframe file has been confirmed written.
- Halt on deferred findings: Deferred findings are normal. The skill completes successfully and documents deferrals in the changelog.
- Mismatch DESIGN-STATE column names: The ux/DESIGN-STATE.md Open Critique Items table uses `Item, Source, Severity, Status` columns (from design-state-domain.md template). Do not use `ID`.
- Look for What Works in templates/critique-report.md: The template does NOT include the `## What Works` section. Read it from the live generated `CRT-critique-v{N}.md` file.
- Apply all findings regardless of effort: If a finding's effort is "significant" and its suggestion requires structural redesign, defer it. Iterate is for targeted corrections, not architectural changes.
- Skip lock-release in Step 5e: always release the lock even on error.
- Assume wireframe version and changelog version are synchronized: They are independent counters — glob for both independently per Step 2e and 2f.
