<objective>
Integrate the sharding library into PDE's plan-phase and execute-phase workflows. After the planner produces PLAN.md files, plan-phase.md automatically shards large plans into task files. During execution, execute-phase.md spawns per-task executors when task files exist, achieving the ~90% context reduction goal.

Purpose: PLAN-01 requires the planner workflow to emit task files; PLAN-02 requires the executor to load only the current task file. This plan wires the sharding engine (built in Plan 01) into the two orchestrator workflows.

Output: Updated plan-phase.md with sharding step, updated execute-phase.md with conditional per-task execution loop.
</objective>
