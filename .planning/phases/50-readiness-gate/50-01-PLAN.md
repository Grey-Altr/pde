    **bin/lib/readiness.cjs:**
    CommonJS module exporting: `runStructuralChecks`, `classifyResult`, `writeReadinessMd`, `cmdReadinessCheck`, `cmdReadinessResult`.

    `runStructuralChecks(planContent, requirementsContent, planFileName)`:
    - Takes plan content string, requirements content string, and plan filename (for error messages)
    - Returns `{ checks: CheckResult[] }` where CheckResult is `{id, description, passed, severity, details}`
    - Category A checks (presence):
      - A1: planContent is truthy (not empty/null). severity: 'fail'
      - A2: extractFrontmatter(planContent).phase exists. severity: 'fail'
      - A3: extractFrontmatter(planContent).requirements is array with length > 0. severity: 'concerns'
      - A4: planContent contains `<objective>`. severity: 'fail'
      - A5: extractPlanAcBlock(planContent) returns non-empty string. severity: 'concerns'
      - A6: extractTaskBlocks(planContent).length > 0. severity: 'fail'
      - A7: For each task block, extractField(inner, 'name') AND extractField(inner, 'action') AND extractField(inner, 'acceptance_criteria') must be non-empty. severity: 'fail'. Details lists which tasks fail.
      - A8: For each task block, extractField(inner, 'ac_refs') must be non-empty. severity: 'concerns'
      - A9: extractFrontmatter(planContent).must_haves?.truths is array with length > 0. severity: 'concerns'
    - Category B checks (consistency, only if requirementsContent provided):
      - B1: Each ID in fm.requirements array must match pattern `**{ID}**` in requirementsContent, AND must NOT be only in the `## Future Requirements` section (use the checkbox presence heuristic: active reqs have `- [ ]` or `- [x]` prefix). severity: 'fail'. Details lists unmapped IDs.
      - B2: Extract AC-N IDs from planAcBlock (regex: `/\*\*AC-(\d+)\*\*:/g`). Extract all ac_refs from task blocks. Each plan AC-N must be referenced by at least one task. severity: 'concerns'. Details lists orphaned AC IDs.
      - B3: Each ac_refs AC-N must exist in planAcBlock AC-N set. severity: 'concerns'. Details lists ghost AC refs.
    - Does NOT check A10/A11 (PROJECT.md/REQUIREMENTS.md existence) — those are filesystem checks done by cmdReadinessCheck, not per-plan-content checks.

    `classifyResult(checks)`:
    - If any check has severity='fail' AND passed=false -> return 'fail'
    - Else if any check has severity='concerns' AND passed=false -> return 'concerns'
    - Else -> return 'pass'

    `writeReadinessMd(phaseDir, phaseNumber, phaseName, allChecks, result)`:
    - Writes `{phaseDir}/{phaseNumber}-READINESS.md`
    - Frontmatter: phase, generated (ISO timestamp), result, checks_run, checks_failed
    - Body: `# Phase {phaseNumber}: Readiness Report` header
    - Sections: `## Structural Checks` table (A* checks), `## Consistency Checks` table (B* checks), `## Executor Handoff` with result summary
    - Table columns: Check | Status | Details
    - Status column: PASS or FAIL (for failing checks) or CONCERNS (for failing concerns-severity checks)

    `cmdReadinessCheck(cwd, phaseArg, planFile, raw)`:
    - Resolve phase directory using findPhaseInternal from core.cjs
    - If planFile specified: check only that plan. Otherwise: glob all *-PLAN.md in phase dir
    - Read .planning/REQUIREMENTS.md
    - Check A10 (PROJECT.md exists) and A11 (REQUIREMENTS.md exists) as filesystem checks
    - For each plan file: read content, call runStructuralChecks(content, reqContent, filename)
    - Aggregate all checks across all plans
    - Call classifyResult on aggregated checks
    - Call writeReadinessMd to write output
    - Output JSON: `{result, file, checks_run, checks_failed, plan_count}`

    `cmdReadinessResult(cwd, phaseArg, raw)`:
    - Resolve phase directory
    - Look for *-READINESS.md in phase dir
    - If found: read frontmatter, return `{result, file, generated}`
    - If not found: return `{result: 'none', file: null}`
