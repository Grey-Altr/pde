The file MUST contain all 5 lint-required sections: `<purpose>`, `<skill_code>`, `<skill_domain>`, `<context_routing>`, `<process>`, plus `<required_reading>`, `<flags>`, and `<output>`.

Structure the workflow as follows:

1. **`<purpose>`**: Discover and recommend MCP servers, tools, and integrations tailored to the user's project context. Reads PROJECT.md, REQUIREMENTS.md, package.json, STACK.md. Produces REC-recommendations-v{N}.md in .planning/design/strategy/. Consumed by /pde:ideate (Phase 27) via Skill() invocation.

2. **`<skill_code>`**: REC

3. **`<skill_domain>`**: strategy

4. **`<context_routing>`**: Detect PROJECT.md (hard requirement), REQUIREMENTS.md (soft), package.json (soft), any STACK.md in .planning/codebase/ (soft), existing DESIGN-STATE (soft).

5. **`<required_reading>`**: Reference @references/skill-style-guide.md and @references/mcp-integration.md

6. **`<flags>`**: Support `--dry-run`, `--quick`, `--verbose`, `--no-mcp`, `--no-compass`, `--no-websearch`, `--force`

7. **`<process>`**: Implement a 7-step pipeline:

   **Step 1/7: Initialize design directories** — `pde-tools.cjs design ensure-dirs` (exact pattern from brief.md Step 1)

   **Step 2/7: Check prerequisites** — Read PROJECT.md (hard fail if missing). Optionally read REQUIREMENTS.md, package.json, .planning/codebase/STACK.md. Check for existing REC artifact and determine version N. If exists and no --force, prompt user.

   **Step 3/7: Probe MCP capabilities** — Check --no-mcp flag first (disables all probes). Then:
   - Probe mcp-compass (targeted MCP): attempt search with project stack keywords; if unavailable, set MCP_COMPASS_AVAILABLE=false and tag `[Using offline catalog -- install mcp-compass for live MCP registry search]`
   - Probe WebSearch MCP: attempt search for "MCP servers for {stack}"; if unavailable, set WEBSEARCH_AVAILABLE=false and tag `[Using training knowledge -- install WebSearch MCP for live tool discovery]`
   - Probe Sequential Thinking MCP: if available, use for enhanced analysis; if unavailable, continue without

   **Step 4/7: Analyze project context and build recommendations** — This is the core logic:
   - Extract from PROJECT.md: product type, tech stack, goals, constraints
   - Extract from REQUIREMENTS.md: feature areas, integration needs
   - Extract from package.json: existing dependencies
   - Match project needs against MCP categories from mcp-integration.md: design, code quality, data, deployment, research, AI, collaboration
   - **Inline curated catalog** (do NOT reference ecosystem-catalog.json — it does not exist): Embed a structured catalog of well-known MCP servers organized by the 7 categories. Include for each entry: name, description, install command (npx/npm), category, relevance signals. This catalog is the fallback when mcp-compass and WebSearch are unavailable. Cover at minimum: Sequential Thinking, WebSearch, filesystem, GitHub, database connectors, Figma, Sentry, Playwright, Context7, and other well-known MCPs.
   - Rank recommendations by relevance to project context (stack match, goal alignment, category coverage)
   - For each recommendation: name, category, relevance score (high/medium/low), why it matches, install command, configuration hint

   **Step 5/7: Write recommendations artifact** — Use the template structure from templates/recommendations.md. Write to `.planning/design/strategy/REC-recommendations-v{N}.md` with YAML frontmatter (Generated, Skill, Version, Status, Scope, Data Currency, Enhanced By). Include sections: Executive Summary, Project Context Analysis, Recommended Tools (ranked), Installation Guide, Integration Notes, Category Coverage Map.

   **Step 6/7: Update domain DESIGN-STATE** — Check for .planning/design/strategy/DESIGN-STATE.md; create from domain template if absent. Add/update REC artifact row.

   **Step 7/7: Update root DESIGN-STATE and manifest** —
   - `lock-acquire pde-recommend`
   - Update root DESIGN-STATE: add/update cross-domain map entry, decision log entry, iteration history
   - `lock-release`
   - `manifest-update REC code REC`, `manifest-update REC name "Tool & MCP Recommendations"`, `manifest-update REC type recommendations`, `manifest-update REC domain strategy`, `manifest-update REC path ".planning/design/strategy/REC-recommendations-v{N}.md"`, `manifest-update REC status draft`, `manifest-update REC version {N}`
   - Coverage flag: `coverage-check` to read all 13 flags, then `manifest-set-top-level designCoverage '{full 13-field JSON with hasRecommendations:true}'` using the pass-through-all pattern

8. **Summary table** and **Anti-Patterns** sections at the end of `<process>`:

9. **`<output>`**: List files written (REC-recommendations-v{N}.md, DESIGN-STATE updates, manifest updates)
