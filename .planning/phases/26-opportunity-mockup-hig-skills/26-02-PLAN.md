Applies design tokens from tokens.css, adds CSS-only interactive states (:hover, :focus-visible, :active, :checked pseudo-classes), preserves wireframe annotations as HTML comments for handoff traceability. Consumed by /pde:critique and /pde:handoff.

7. **`<process>`** -- 7-step pipeline:

   **Step 1/7: Initialize design directories** -- `pde-tools.cjs design ensure-dirs` (creates ux/mockups/ among others)

   **Step 2/7: Check prerequisites and discover wireframes** -- Use Glob to find `.planning/design/ux/wireframe-*-v*.html`. For each screen, identify the highest version. Read design system artifact for token values. Read brief for product context. Check for existing mockup files and determine version N. If wireframes absent: warn but continue (soft dependency). If --screen flag: filter to single screen.

   **Step 3/7: Probe MCP availability** -- Check --no-mcp flag. Probe Playwright MCP: if available, can preview generated HTML for visual verification. If unavailable, continue without preview. No other MCPs needed for mockup generation.

   **Step 4/7: Generate hi-fi HTML/CSS mockups** -- For each wireframe screen:

   a. Read the wireframe HTML source file completely.
   b. Extract all `<!-- WIREFRAME-ANNOTATION: ... -->` comments and preserve them.
   c. Generate the mockup HTML following this exact structure:

   ```html
   <!DOCTYPE html>
   <html lang="en">
   <head>
     <meta charset="UTF-8">
     <meta name="viewport" content="width=device-width, initial-scale=1.0">
     <title>{Screen Name} -- {Product Name} Mockup</title>
     <!-- WIREFRAME-SOURCE: wireframe-{screen}-v{N}.html | Generated: {date} -->
     <link rel="stylesheet" href="../../assets/tokens.css">
     <style>
       /* All custom styles inline -- self-contained per MOCK-01 */
       /* CSS-only interactive states -- no JavaScript per MOCK-02 */
       .btn-primary:hover { background: var(--color-primary-600); }
       .btn-primary:focus-visible {
         outline: 3px solid var(--color-focus-ring);
         outline-offset: 2px;
       }
       .btn-primary:active { transform: scale(0.98); }
       /* ... additional component styles using design tokens ... */
     </style>
   </head>
   <body class="pde-layout--hifi" data-screen="{screen-slug}" data-version="{N}">
     <!-- WIREFRAME-ANNOTATION: {preserved annotation text from wireframe} -->
     <!-- ... evolved hi-fi HTML content ... -->
   </body>
   </html>
   ```

   Key rules for HTML generation:
   - ONLY external dependency: `../../assets/tokens.css` (same relative path as wireframe)
   - ALL custom CSS in `<style>` block (no external CSS bundle -- mockup.css does NOT exist)
   - NO `<script>` tags, NO JavaScript event listeners, NO onclick attributes (MOCK-02: CSS-only)
   - Interactive states via CSS pseudo-classes only: `:hover`, `:focus-visible`, `:active`, `:checked`, `:disabled`, `[open]` for details/summary
   - Use CSS custom properties from tokens.css: `var(--color-*)`, `var(--space-*)`, `var(--font-*)`, `var(--radius-*)`, `var(--shadow-*)`
   - Body class `pde-layout--hifi` (wireframe uses pde-layout--{fidelity})
   - First comment in `<head>`: `<!-- WIREFRAME-SOURCE: wireframe-{screen}-v{N}.html | Generated: {date} -->`
   - Every wireframe `<!-- WIREFRAME-ANNOTATION: ... -->` comment preserved in corresponding location in mockup HTML
   - Responsive: use CSS Grid/Flexbox, media queries for mobile/tablet/desktop breakpoints
   - Accessibility: all interactive elements have focus-visible styles, proper ARIA roles, semantic HTML

   d. Write each screen to `.planning/design/ux/mockups/mockup-{screen}.html`

   e. Generate `index.html` in `.planning/design/ux/mockups/` -- navigation page linking all generated screen mockups. Self-contained HTML with inline styles.

   **Step 5/7: Write mockup specification artifact** -- Use template from templates/mockup-spec.md (ignore mockup.js/mockup.css references). Write to `.planning/design/ux/mockups/MCK-mockup-spec-v{N}.md` with YAML frontmatter. Document: screens generated, token mappings applied, interaction states implemented, wireframe annotations preserved.

   **Step 6/7: Update domain DESIGN-STATE** -- Check for `.planning/design/ux/DESIGN-STATE.md`; create from domain template if absent. Add/update MCK artifact row.

   **Step 7/7: Update root DESIGN-STATE and manifest** --
   - `lock-acquire pde-mockup`

   - Coverage: `coverage-check` to read all 13 flags, then `manifest-set-top-level designCoverage` with full 13-field JSON setting `hasMockup: true` and all others preserved

8. **`<output>`**: mockup-{screen}.html per screen, index.html, MCK-mockup-spec-v{N}.md, DESIGN-STATE updates, manifest updates
