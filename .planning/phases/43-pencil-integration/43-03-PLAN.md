**Purpose tag:**
```xml
<purpose>
Capture a Pencil canvas screenshot for inclusion in /pde:critique visual audit. Non-blocking sub-workflow: if Pencil is not connected or the screenshot fails, returns immediately with no output (critique continues with existing wireframe files). Writes screenshot to .planning/design/ux/wireframes/pencil-canvas.png for discovery by critique Step 2d wireframe glob. Implements PEN-02.
</purpose>
```

**Step 0 — Check Pencil Connection:**
```javascript
// bash block (ESM + createRequire pattern)
import { createRequire } from 'module';
const req = createRequire(import.meta.url);
const b = req(`${process.env.CLAUDE_PLUGIN_ROOT}/bin/lib/mcp-bridge.cjs`);
const conn = b.loadConnections();
const pen = conn.connections && conn.connections.pencil;
const status = pen && pen.status || 'not_configured';
let screenshotToolName = '';
try {
  screenshotToolName = b.call('pencil:get-screenshot', {}).toolName;
} catch (err) {
  screenshotToolName = '';
}
process.stdout.write(JSON.stringify({ status, screenshotToolName }) + '\n');
```

If `status` != 'connected' OR `screenshotToolName` is empty:
Display: `Pencil not connected — skipping canvas screenshot. Critique will use existing wireframe files. Run /pde:connect pencil to enable.`
Return immediately. This is a NON-BLOCKING sub-workflow — critique continues.

**Step 1 — Capture Screenshot via MCP:**
This is a Claude Code tool call (NOT inside bash block). Instruct Claude:
"Use the tool `{screenshotToolName}` (i.e., `mcp__pencil__get_screenshot`) to capture a screenshot of the current Pencil canvas."

Handle the response adaptively:
- If response contains image content (base64-encoded): proceed to Step 2
- If response has `data:image/` prefix: strip it before decoding
- If response is empty, null, or unrecognizable format: log warning `"Pencil screenshot response could not be decoded. Continuing critique without canvas screenshot."` and return (non-blocking)

**Step 2 — Write Screenshot File:**
```javascript
// bash block — decode base64 and write PNG
import { createRequire } from 'module';
const req = createRequire(import.meta.url);
const fs = req('fs');
const path = req('path');
const outputDir = path.join(process.cwd(), '.planning', 'design', 'ux', 'wireframes');
fs.mkdirSync(outputDir, { recursive: true });
// SCREENSHOT_BASE64 is set from the MCP response in Step 1
const buffer = Buffer.from(process.env.SCREENSHOT_BASE64, 'base64');
fs.writeFileSync(path.join(outputDir, 'pencil-canvas.png'), buffer);
process.stdout.write('pencil-canvas.png written to ' + outputDir + '\n');
```

If write fails: log warning and continue (non-blocking).

**Step 3 — Return Path:**
Display: `Pencil canvas screenshot captured: .planning/design/ux/wireframes/pencil-canvas.png`
Display: `This file will be included in the critique visual evaluation.`

The critique pipeline (Step 2d) discovers this file via its wireframe glob. No explicit wiring needed — file presence is sufficient.

**Important degraded mode notes (PEN-03):**
- Every failure path in this workflow returns without blocking
- No `process.exit(1)` anywhere — only `process.stdout.write` for messaging
- The critique pipeline MUST NOT depend on this sub-workflow succeeding
- If Pencil is unavailable, critique uses existing wireframe HTML files (existing behavior unchanged)
