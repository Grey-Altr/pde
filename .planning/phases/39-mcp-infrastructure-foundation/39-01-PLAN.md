4. **CONNECTIONS_PATH** — `path.join(process.cwd(), '.planning', 'mcp-connections.json')`

5. **assertApproved(serverKey)** — throws an Error with `code: 'POLICY_VIOLATION'` if serverKey is not in APPROVED_SERVERS. Error message must include: the rejected key name, the phrase "is not an approved MCP server", and the full list of approved server keys joined by ", ".

6. **loadConnections()** — reads CONNECTIONS_PATH via `safeReadFile`. Returns `{ schema_version: '1.0', connections: {} }` if file missing or parse fails. Never throws.

7. **saveConnections(data)** — writes data as pretty-printed JSON (2-space indent) to CONNECTIONS_PATH. Creates `.planning/` directory if needed via `fs.mkdirSync(dir, { recursive: true })`.

8. **getStatus(serverKey)** — calls `assertApproved(serverKey)`, reads connections via `loadConnections()`, returns the connection entry for that key or `{ status: 'not_configured', server_key: serverKey, display_name: APPROVED_SERVERS[serverKey].displayName }` if not found.

9. **getAllStatuses()** — iterates all keys in APPROVED_SERVERS, calls `getStatus` for each, returns an object keyed by server key.

10. **probe(serverKey)** — calls `assertApproved(serverKey)`. Checks if `APPROVED_SERVERS[serverKey].probeTool` is null — if so, returns `{ available: false, status: 'not_configured', reason: 'probe_not_implemented' }`. Otherwise returns `{ available: false, status: 'probe_deferred', reason: 'Probe tool calls require Claude Code MCP runtime — use within workflow context' }`. This function is a coordination point; actual MCP tool calls happen at the workflow level where Claude Code MCP tools are available.

11. **call(canonicalName, args)** — looks up `canonicalName` in TOOL_MAP. If not found, throws Error with message `Tool "${canonicalName}" not found in TOOL_MAP. Available: ${Object.keys(TOOL_MAP).join(', ') || 'none (populate in phases 40-44)'}`. If found, returns `{ toolName: TOOL_MAP[canonicalName], args }` (the actual MCP call is made by the workflow layer via Claude Code). This is a lookup/validation layer, not an execution layer.

12. **updateConnectionStatus(serverKey, status, extraFields)** — calls `assertApproved(serverKey)`, loads connections, merges `{ server_key: serverKey, display_name: APPROVED_SERVERS[serverKey].displayName, transport: APPROVED_SERVERS[serverKey].transport, url: APPROVED_SERVERS[serverKey].url, status, ...extraFields, last_updated: new Date().toISOString() }` into `connections[serverKey]`, saves connections. Returns the updated entry.

File must start with `'use strict';` and end with `module.exports = { ... }` exporting all public functions and constants listed above.

IMPORTANT: Do NOT include any fields named `token`, `secret`, `key`, `password`, or `auth` in mcp-connections.json data structures. The only "auth" reference allowed is in AUTH_INSTRUCTIONS (user-facing text).

IMPORTANT: Do NOT attempt to call MCP tools directly from this module. Claude Code is the MCP runtime; this module is a coordination/policy layer.
