**Create `workflows/mcp-status.md`:**

The workflow must implement this exact process:

## 0. Initialize
- Parse $ARGUMENTS for `--json` flag (boolean, default false).

## 1. Load Status Data
- Run: `node -e "const b = require('${CLAUDE_PLUGIN_ROOT}/bin/lib/mcp-bridge.cjs'); console.log(JSON.stringify(b.getAllStatuses()))"`
- Parse the JSON output into a `statuses` object.

## 2. Load Connection Metadata
- Run: `node -e "const b = require('${CLAUDE_PLUGIN_ROOT}/bin/lib/mcp-bridge.cjs'); console.log(JSON.stringify(b.loadConnections()))"`
- Parse the JSON output. This gives `last_probe_at`, `connected_at` fields if they exist.

## 3. Display Status

If `--json` flag: output the raw statuses JSON and stop.

Otherwise, format a status table. For each server key in APPROVED_SERVERS (github, linear, figma, pencil, atlassian), display one row with:
- Server display name (left-aligned, padded to 15 chars)
- Status: one of `connected`, `disconnected`, `degraded`, `not_configured`, shown as `--` for not_configured
- Hint: if not_configured, show `Run /pde:connect <key> to set up`. If disconnected, show `Run /pde:connect <key> to reconnect`. If connected, show last probe time if available.

Output header: `MCP Integration Status` followed by a blank line.

Output footer: `{N}/5 integrations active. Run /pde:connect <name> to add.` where N is the count of entries with status `connected`.

Example output format:
```
MCP Integration Status

  GitHub           connected      Last probe: 2 min ago
  Linear           disconnected   Run /pde:connect linear to reconnect
  Figma            --             Run /pde:connect figma to set up
  Pencil           --             Run /pde:connect pencil to set up
  Atlassian        --             Run /pde:connect atlassian to set up

1/5 integrations active. Run /pde:connect <name> to add.
```

## 4. Suggest Next Steps
- If 0 integrations active: suggest running `/pde:connect github` to start with GitHub (most common integration).
- If all 5 active: note all integrations are connected.

The workflow must reference `@${CLAUDE_PLUGIN_ROOT}/references/mcp-integration.md` in required_reading for the global `--no-mcp` flag handling. If `--no-mcp` is present in arguments, skip all probes and show all statuses from stored metadata only (no live probe attempts).

**Create `workflows/connect.md`:**

The workflow must implement this exact process:

## 0. Initialize
- Parse $ARGUMENTS to extract:
  - `SERVICE_KEY`: first positional argument (required). If missing, display help: `Usage: /pde:connect <service>` followed by the list of approved servers from APPROVED_SERVERS and stop.
  - `--confirm` flag: boolean, default false.
  - `--disconnect` flag: boolean, default false.
- Convert SERVICE_KEY to lowercase and trim.

## 1. Validate Server
- Run: `node -e "const b = require('${CLAUDE_PLUGIN_ROOT}/bin/lib/mcp-bridge.cjs'); b.assertApproved('${SERVICE_KEY}'); console.log('APPROVED')"`
- If this throws (exit code non-zero), the error output will contain the policy violation message. Display that message to the user verbatim and stop. Do NOT proceed.
- The policy message must include: the rejected key, the phrase "is not an approved MCP server", and the list of approved servers.

## 2. Handle --disconnect
- If `--disconnect` flag is present:
  - Run: `node -e "const b = require('${CLAUDE_PLUGIN_ROOT}/bin/lib/mcp-bridge.cjs'); const r = b.updateConnectionStatus('${SERVICE_KEY}', 'disconnected', { disconnected_at: new Date().toISOString() }); console.log(JSON.stringify(r))"`
  - Display: `Disconnected from ${displayName}. Connection metadata updated.`
  - Suggest: `Run 'claude mcp remove ${SERVICE_KEY}' to also remove the MCP server registration.`
  - Stop.

## 3. Display Auth Instructions
- If `--confirm` flag is NOT present:
  - Run: `node -e "const b = require('${CLAUDE_PLUGIN_ROOT}/bin/lib/mcp-bridge.cjs'); console.log(JSON.stringify(b.AUTH_INSTRUCTIONS['${SERVICE_KEY}']))"`
  - Parse the JSON array of instruction strings.
  - Display a header: `Connecting to ${displayName}` followed by a blank line.
  - Display each instruction string on its own line.
  - After instructions, display: `After completing the steps above, run: /pde:connect ${SERVICE_KEY} --confirm`
  - Stop. Do not proceed further without --confirm.

## 4. Confirm Connection (--confirm present)
- Run: `node -e "const b = require('${CLAUDE_PLUGIN_ROOT}/bin/lib/mcp-bridge.cjs'); const r = b.updateConnectionStatus('${SERVICE_KEY}', 'connected', { connected_at: new Date().toISOString() }); console.log(JSON.stringify(r))"`
- Display: `Connected to ${displayName}. Status saved to .planning/mcp-connections.json.`
- Display: `Run /pde:mcp-status to see all connection states.`

## 5. Error Handling
- If any node command fails unexpectedly, display the error and suggest checking that `bin/lib/mcp-bridge.cjs` exists and loads cleanly.
- Never crash without a user-facing message.

The workflow must reference `@${CLAUDE_PLUGIN_ROOT}/references/mcp-integration.md` in required_reading.

IMPORTANT: The connect workflow does NOT attempt to call any MCP tools itself. It guides the user through auth setup (which happens outside PDE via `claude mcp add` and `/mcp`) and then records the connection metadata. The actual MCP server connection is managed by Claude Code's runtime.
