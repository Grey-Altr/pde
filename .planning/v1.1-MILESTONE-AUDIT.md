### Integration Issues

#### INT-01 (HIGH): hasBrief Coverage Flag Contradiction

`workflows/brief.md` anti-pattern section explicitly bans setting `hasBrief` in `designCoverage`. The manifest schema (template and live) omits `hasBrief`. However, `workflows/system.md` Step 7 instructs "preserve `hasBrief` if already true" — this instruction is permanently dead because the field can never be set.

**Fix:** Remove `hasBrief` references from `system.md` Step 7 (lines ~1326, 1330). Also check `flows.md` Step 7 for the same pattern.

**Affected:** BRF-01, BRF-02, SYS-01, SYS-02, SYS-03

#### INT-02 (MEDIUM): Domain DESIGN-STATE Table Stubs

Root `DESIGN-STATE.md` Domain Files table is pre-populated with rows for all 6 domains, each showing `{count}` as artifact count and a fixed date. These are never substituted — they remain literal placeholder strings. Domain DESIGN-STATE files don't exist until their first skill run.

**Fix:** Either: (a) Remove pre-populated rows from `templates/design-state-root.md`, let each skill append its row on first run; or (b) Replace `{count}` with `0` and mark date as `--` until first skill run.

**Affected:** INFRA-01, INFRA-02

#### INT-03 (LOW): Orphaned CLI Subcommands

`design tokens-to-css` and `design artifact-path` are routed in `pde-tools.cjs` and implemented in `design.cjs`, but no Phase 12-15 workflow calls them. `system.md` generates CSS inline via Write tool instead of calling the CLI. Expected consumers are in Phases 16-20.

**Impact:** Minimal — forward-facing capability. INFRA-03 is satisfied at the library level.

### Broken Flows

1. **hasBrief coverage flow:** Brief → coverage-check → system.md preserve → dead instruction (INT-01)
2. **Domain Files table:** ensureDesignDirs → root DESIGN-STATE → stale placeholder data (INT-02)
