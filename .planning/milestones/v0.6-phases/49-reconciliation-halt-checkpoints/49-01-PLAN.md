**File content must include these sections in order:**

1. `<purpose>` — "Gather execution evidence from git history, compare against planned tasks, and produce RECONCILIATION.md."

2. `<required_reading>` — Reference STATE.md and project-context.md.

3. `<process>` with these steps:

**Step 1 — collect_planned_tasks:**
- Read all PLAN.md files in the phase directory (passed via `{phase_dir}` input)
- For each task in each plan: extract task name, files list, ac_refs
- Build an array of `{plan_id, task_name, task_files, ac_refs}` objects

**Step 2 — collect_git_evidence:**
- Run `git log --oneline --grep="({phase_number}-" --all` to get all phase commits (parenthesis-anchored to avoid cross-phase matches per Pitfall 5)
- For each commit, run `git show --stat {hash} --name-only` to get changed files
- Build an array of `{hash, message, files}` objects
- Run `git diff --name-only {PHASE_FIRST_COMMIT}^..HEAD` to get all files changed since phase start

**Step 3 — match_tasks_to_commits:**
Task-to-commit matching algorithm (three tiers):

```
Primary: slug matching
- Normalize task name to slug: lowercase, replace non-alphanumeric with hyphens, remove leading/trailing hyphens
- Normalize commit message (text after "): ") to slug with same rules
- Match if they share 3+ consecutive words (split slug by hyphens)

Fallback: file overlap
- If slug match fails, check if any file in the commit's changed files appears in the task's <files> list
- If overlap found: assign commit to that task

Weak match: phase-plan prefix
- If both slug and file overlap fail, check if commit message contains "({phase_number}-{plan_number})"
- If yes: mark as "likely associated" with the plan (not a specific task)
- If no: classify as "unplanned"
```

**Step 4 — derive_ac_status:**
For each task with ac_refs:
- If task has a matching commit: `likely satisfied`
- If task has a matching commit but SUMMARY.md lists deviations for that task: `completed with deviation — verify`
- If no matching commit found: `not executed`

Read SUMMARY.md deviation data from `{phase_dir}/*-SUMMARY.md`.

**Step 5 — detect_unplanned_changes:**
Compare all files from `git diff --name-only` against the union of all task `<files>` lists.
Files not in any task list: classified as unplanned changes.
For each unplanned file, note the commit that introduced it and assess: `minor support file` (test helpers, config tweaks, type fixes) or `potentially significant` (new feature files, schema changes).

**Step 6 — determine_status:**
```
if any planned task has no matching commit: status = "incomplete"
else if any task has deviation in SUMMARY.md: status = "deviations_found"
else if any unplanned files detected: status = "unplanned_changes"
else: status = "clean"
```
Precedence: incomplete > deviations_found > unplanned_changes > clean.
