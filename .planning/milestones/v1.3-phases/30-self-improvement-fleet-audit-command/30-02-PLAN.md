- `--category "X,Y"`: comma-separated category filter (default: all)

## 1. Spawn Auditor Agent

Build the artifact list by scanning the PDE directory:

```bash
COMMANDS=$(ls "${CLAUDE_PLUGIN_ROOT}"/commands/*.md 2>/dev/null | tr '\n' ',')

TEMPLATES=$(find "${CLAUDE_PLUGIN_ROOT}"/templates -name "*.md" 2>/dev/null | tr '\n' ',')
REFERENCES=$(ls "${CLAUDE_PLUGIN_ROOT}"/references/*.md 2>/dev/null | tr '\n' ',')

If `--category` flag is set, filter to only the specified categories.

Spawn the auditor Task:

Task(

## 3. Baseline Handling

If `--save-baseline` flag is set:
- Write `.planning/audit-baseline.json` with the scores object from the auditor, plus timestamp and finding_count

If `.planning/audit-baseline.json` exists (regardless of flag):
- Load the baseline
- Compute delta: overall_delta = current overall_health_pct - baseline overall_health_pct
- Include delta in the audit report's "Delta from Baseline" section

## 4. Improvement Loop (only if --fix flag)

If `--fix` is NOT set: skip to Step 5.

Filter findings to CRITICAL and HIGH severity. If `--deep` is set, include MEDIUM and LOW too.

Group findings by artifact file. For each artifact group (up to 10 artifacts max per run):

### 4a. Spawn Improver

```
Task(
  prompt="Read ${CLAUDE_PLUGIN_ROOT}/agents/pde-skill-improver.md for instructions.

<findings>
{JSON array of findings for this artifact}
</findings>

<original_artifact_path>{artifact_path}</original_artifact_path>
<staging_path>.planning/improvements/</staging_path>
<protected_files_path>${CLAUDE_PLUGIN_ROOT}/protected-files.json</protected_files_path>",
  model="{IMPROVER_MODEL}"
)
```

Parse the improver's return JSON. Record proposal_path.

### 4b. Spawn Validator

```
Task(
  prompt="Read ${CLAUDE_PLUGIN_ROOT}/agents/pde-skill-validator.md for instructions.

<proposal_path>{proposal_path}</proposal_path>
<original_artifact_path>{artifact_path}</original_artifact_path>",
  model="{VALIDATOR_MODEL}"
)
```

Parse the validator's return JSON.

### 4c. Handle Validation Result

- If `status: "PASS"` and `recommendation: "apply"`:
  - Check if the target artifact path is in protected-files.json `protected[]` or under `protected_directories[]`
  - If protected: log "SKIPPED (protected file)" — do not apply
  - If not protected: read the proposal's "Proposed Section", apply the change to the original artifact file
  - Log: "APPLIED: {artifact} ({rule})"

- If `status: "FAIL"` and `recommendation: "revise"`:
  - Re-spawn improver with the validator's issues as additional context
  - Re-validate (max 3 total cycles per artifact — cap at 3 to prevent infinite loops)
  - On 3rd FAIL: log "NEEDS HUMAN REVIEW: {artifact} ({rule})" and move on

- If `recommendation: "reject"`:
  - Log "REJECTED: {artifact} ({rule}) — {reason}" and move on

### 4d. Update Audit Report

Append an "## Improvements Applied" section to `.planning/audit-report.md` listing:
- Applied changes with artifact path and rule
- Skipped (protected) changes
- Rejected changes
- Human review items

## 5. Display Summary

Print to stdout:
```
PDE Audit Complete
  Findings: {total} ({critical} critical, {high} high, {medium} medium, {low} low)
  Health:   {overall_pct}%
  Delta:    {+/-N.N}% from baseline ({baseline_date}) | "no baseline"
  Applied:  {N} improvements | "audit-only mode (use --fix to apply)"
  Report:   .planning/audit-report.md
```

</process>
```

CRITICAL IMPLEMENTATION NOTES:
- The workflow MUST drive the loop. Agents do NOT spawn other agents. Each Task() call is sequential — wait for return before spawning next.
- The protected-files guard in Step 4c is the WORKFLOW's responsibility, not just the agent's. Double-check before applying.
- Cap improvement cycles at 3 per artifact to prevent infinite loops (Pitfall 4 from research).
- Cap total artifacts improved at 10 per run to prevent context exhaustion (Pitfall 4 from research).
- Group findings by artifact — one improver Task per artifact, not one per finding.
