7. **`<process>`** — 7-step pipeline:

   **Step 1/7: Initialize design directories** — `pde-tools.cjs design ensure-dirs`

   **Step 2/7: Check prerequisites and discover candidates** — Read PROJECT.md (hard fail if missing). Use Glob to find `.planning/design/strategy/CMP-competitive-v*.md`, sort descending by version, read highest. If found: parse `## Opportunity Highlights` > `### Top Opportunities` numbered list. Pre-populate candidates table and present to user: "Found {N} opportunities from /pde:competitive gap analysis. Review and confirm before scoring (add/remove/modify as needed)." If not found: start with empty list, ask user for initial candidates. Check for existing OPP artifact version, determine N. If exists and no --force: prompt user to confirm overwrite.

   **Step 3/7: Probe MCP availability** — Check --no-mcp flag first. Probe Sequential Thinking MCP: if available, use for enhanced analysis of scoring calibration. If unavailable, continue without. No other MCPs needed for opportunity scoring (WebSearch and mcp-compass not used here).

   **Step 4/7: Interactive RICE scoring with design extensions** — For each confirmed candidate, collect interactively:
   1. "Reach: How many users affected in one quarter?" → Accept integer
   2. "Impact: How much does this move the needle per user? [3=Massive / 2=High / 1=Medium / 0.5=Low / 0.25=Minimal]" → Accept 3, 2, 1, 0.5, 0.25
   3. "Confidence: How sure are you about these estimates? [1.0=High (data) / 0.8=Medium (some data) / 0.5=Low (gut feel)]" → Accept 1.0, 0.8, 0.5
   4. "Effort: T-shirt size estimate? [XS=0.5mo / S=1mo / M=2mo / L=4mo / XL=8mo]" → Accept label, map to numeric months
   5. "UX Differentiation: Design advantage over competitors? [3=Breakthrough / 2=Strong / 1=Moderate / 0=None]"
   6. "Accessibility Impact: [3=Critical barrier removed / 2=Significant / 1=Moderate / 0=None]"
   7. "Design System Leverage: [3=Creates 5+ reusable components / 2=Moderate / 1=Low / 0=None]"

   After each candidate: compute immediately and show preview:
   ```
   RICE_base = (Reach * Impact * Confidence) / Effort
   Design_bonus = (UX_diff * 0.5) + (A11y_impact * 0.3) + (DS_leverage * 0.2)
   Final_score = RICE_base * (1 + Design_bonus / 10)
   ```

   After all candidates: show ranked table sorted by Final_score descending.

   Then compute sensitivity analysis (skip if --quick):
   - For each candidate, compute 3 scenarios: Optimistic (Confidence +0.2, capped at 1.0), Pessimistic (Confidence -0.2, floor 0.3), Effort Doubled.
   - For EACH scenario: recompute ALL candidates' adjusted scores, sort ALL candidates by adjusted score, compare to original ranking, output rank change as +N/-N/-- (not just score change).
   - Narrative: identify "fragile" items (rank drops >= 2 with small perturbation) and "robust" items (rank stable across all scenarios).

   Then assign Now/Next/Later buckets per template criteria:
   - Now: top 30% by Final_score AND Effort <= 2 months AND no blockers
   - Next: top 60% by Final_score OR Effort > 2 months
   - Later: remaining AND items flagged as "fragile" in sensitivity analysis

   **Step 5/7: Write opportunity artifact** — Use template from templates/opportunity-evaluation.md. Write to `.planning/design/strategy/OPP-opportunity-v{N}.md`. Include YAML frontmatter (Generated, Skill: /pde:opportunity (OPP), Version, Status: draft, Scope, Data Currency). Include sections: Executive Summary, Scoring Methodology, Candidates Table (all 7 dimensions + computed scores), Sensitivity / Scenario Models (with Rank Change column), Now/Next/Later Roadmap Buckets, Assumptions and Caveats.

   **Step 6/7: Update domain DESIGN-STATE** — Check for `.planning/design/strategy/DESIGN-STATE.md`; create from domain template if absent. Add/update OPP artifact row.

   **Step 7/7: Update root DESIGN-STATE and manifest** —
   - `lock-acquire pde-opportunity`

   - `lock-release`
