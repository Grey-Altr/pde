1. **`<purpose>`**: Audit design artifacts against WCAG 2.2 Level AA criteria and platform Human Interface Guidelines (Apple HIG, Material Design, Web). Produces severity-rated findings covering all ~56 WCAG 2.2 criteria in full mode, or 5 mandatory checks in --light mode for critique delegation. Full mode writes HIG-audit-v{N}.md to .planning/design/review/. Light mode returns inline findings only. Consumed by /pde:critique (--light) and /pde:iterate (full).

2. **`<skill_code>`**: HIG

3. **`<skill_domain>`**: review

4. **`<context_routing>`**:
   - Hard requirement: At least one design artifact to audit (wireframe HTML, mockup HTML, or design system artifact) -- HALT if no auditable artifact found
   - Soft dependencies: `.planning/design/ux/wireframe-*-v*.html`, `.planning/design/ux/mockups/mockup-*.html`, `.planning/design/visual/SYS-system-v*.md`, `.planning/design/strategy/BRF-brief-v*.md`

   - `--verbose` -- Show individual criterion check details
   - `--no-mcp` -- Disable all MCP probes
   - `--no-axe` -- Skip Axe a11y MCP specifically. Fall back to manual WCAG checklist.
   - `--force` -- Overwrite existing HIG artifact without prompting

   **Step 1/7: Initialize design directories** -- `pde-tools.cjs design ensure-dirs`

   **Step 2/7: Check prerequisites and detect mode** --

   - Detect fidelity level of auditable artifacts: mockup HTML = hifi, wireframe --hifi = hifi, wireframe --midfi = midfi, wireframe --lofi = lofi, design system only = lofi. Store as FIDELITY_LEVEL.
   - Scan for auditable artifacts (mockup first, then wireframe, then design system). HALT if none found.
   - If not --light: check for existing HIG artifact version, determine N.
   - Detect target platform from --platform flag (default: web).

   **Step 3/7: Probe MCP availability** -- Check --no-mcp and --no-axe flags. Probe Axe a11y MCP: `mcp__a11y__*` tools. If available: set AXE_AVAILABLE=true, will use for automated WCAG scanning. If unavailable: set AXE_AVAILABLE=false, log "Axe MCP: unavailable. Using manual WCAG checklist from wcag-baseline.md."

   **Step 4/7: Execute WCAG + HIG audit** --

   **IF --light mode (LIGHT_MODE=true):**

   ```
   | Severity | Effort | Location | Issue | Suggestion | Reference |

   ```

   STOP HERE for --light mode. Do NOT proceed to Steps 5-7. Do NOT write any artifact file.

      - 2.4.11 Focus Not Obscured (Minimum)
      - 2.5.7 Dragging Movements
      - 2.5.8 Target Size (Minimum)
      - 3.2.6 Consistent Help
      - 3.3.7 Redundant Entry

   d. **Platform-Specific HIG Checks** (based on --platform flag):

      - ios: Apple HIG patterns (Dynamic Type, SF Symbols, system controls, haptics) from interaction-patterns.md Swift sections
      - android: Material Design patterns (touch ripple, elevation semantics, content descriptions)

   e. **Component-Grouped View** -- Group findings by UI component (navigation, forms, cards, modals, etc.)

   f. **Severity Rating** -- Every finding rated using this EXACT scale (must match critique):
      - critical: Blocks access entirely (form has no labels, element not keyboard-reachable)
      - major: Significant barrier (color contrast fails AA, focus indicator not visible, touch target < 24px)
      - minor: Moderate issue (ARIA label imprecise, heading hierarchy skips one level)
      - nit: Polish (aria-describedby could improve, decorative image alt handling)

Include sections: Executive Summary, POUR Compliance Table, Findings by Severity, Findings by Component, Platform-Specific Notes, WCAG 2.2 New Criteria Results, Recommendations.

   **Step 6/7: Update domain DESIGN-STATE** (SKIP if --light) -- Check for `.planning/design/review/DESIGN-STATE.md`; create if absent. Add/update HIG artifact row.

   **Step 7/7: Update root DESIGN-STATE and manifest** (SKIP if --light) --
