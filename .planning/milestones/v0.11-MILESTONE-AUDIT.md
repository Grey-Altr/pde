## Cross-Phase Integration

### Finding 1 — CRITICAL: designCoverage field clobber

**9 workflows write 14-field designCoverage objects, omitting `hasPrintCollateral` and `hasProductionBible`.**

Affected workflows: `system.md`, `critique.md`, `iterate.md`, `hig.md`, `ideate.md`, `competitive.md`, `opportunity.md`, `recommend.md`, `mockup.md`

Additionally, `wireframe.md` writes 15 fields (missing only `hasProductionBible`).

Only `flows.md` and `handoff.md` write the correct 16-field schema.

**Failure scenario:** User runs `/pde:wireframe` (sets `hasPrintCollateral: true`) → runs `/pde:critique` (writes 14 fields, clobbers `hasPrintCollateral` to false) → runs `/pde:handoff` (reads `hasPrintCollateral: false`, may omit print spec from BIB).

**Fix:** Add `hasPrintCollateral` and `hasProductionBible` fields to the designCoverage write template in all 10 affected workflows (9 with 14 fields + wireframe.md with 15 fields).

**Affected requirements:** HDOF-06, PRNT-01, PRNT-02, PRNT-03, PRNT-04

### Finding 2 — MODERATE: Token file path mismatch

**`SYS-experience-tokens.json` path mismatch between system.md (writer) and wireframe.md (reader).**

- system.md writes to: `.planning/design/visual/SYS-experience-tokens.json`
- wireframe.md reads from: `.planning/design/assets/SYS-experience-tokens.json`

The file is never found. Wireframe falls back to a 3-color default palette. Brand coherence tokens never flow to print artifacts.

**Fix:** Align wireframe.md line 161 to read from `.planning/design/visual/SYS-experience-tokens.json`.

**Affected requirements:** DSYS-06, PRNT-04

### Finding 3 — MINOR: Documentation typo

REQUIREMENTS.md line 46 uses artifact code `FPL` instead of `FLP`. All implementations consistently use `FLP`.

**Fix:** Correct typo in REQUIREMENTS.md.

**Affected requirement:** WIRE-03 (documentation only)
