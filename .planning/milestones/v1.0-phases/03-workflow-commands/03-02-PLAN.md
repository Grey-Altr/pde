<!-- Command stub template pattern from research (Pattern 1) -->
Each command stub must follow this exact structure:

```markdown
---
name: pde:command-name
description: Brief description of what the command does
argument-hint: "<args>"
allowed-tools:
  - Read
  - Write
  - Edit
  - Bash
  - Glob
  - Grep
  - Task
---
<objective>
Execute the /pde:command-name workflow.
</objective>

<execution_context>
@${CLAUDE_PLUGIN_ROOT}/workflows/command-name.md
</execution_context>

<process>
Execute the workflow from @${CLAUDE_PLUGIN_ROOT}/workflows/command-name.md.
Pass any $ARGUMENTS to the workflow process.
</process>
```

IMPORTANT: The `name:` field MUST include the `pde:` prefix. Without it, the command registers as `/command-name` instead of `/pde:command-name`.
