**Header:**
```bash
#!/usr/bin/env bash
# pane-token-meter.sh — live token estimate and cost meter
# TOKN-01: chars/4 heuristic, all values labeled "~est."
# TOKN-02: per-model pricing from model-profiles config
```

**Argument handling:** Same as current stub — `NDJSON="${1:-}"`, exit 1 if empty.

**Model/pricing resolution (runs ONCE at startup via inline Node.js):**
- Resolve `PLUGIN_ROOT` using `CLAUDE_PLUGIN_ROOT` env var or `$(cd "$(dirname "$0")/.." && pwd)`
- Single `node -e` invocation that:
  1. Requires `model-profiles.cjs` from `${PLUGIN_ROOT}/bin/lib/model-profiles.cjs`
  2. Reads `.planning/config.json` to get `model_profile` (default: `balanced`)
  3. Looks up `MODEL_PROFILES['pde-executor'][profile]` to get model tier (default: `sonnet`)
  4. Embeds pricing table inline:
     ```javascript
     const PRICING = {
       opus:   { input: 5.00,  output: 25.00, context_window: 1000000 },
       sonnet: { input: 3.00,  output: 15.00, context_window: 1000000 },
       haiku:  { input: 1.00,  output: 5.00,  context_window:  200000 },
     };
     ```
  5. Outputs pipe-delimited: `modelName|inputPrice|outputPrice|contextWindow`
- Parse output with `cut -d'|'` into `MODEL_NAME`, `INPUT_PRICE`, `OUTPUT_PRICE`, `CONTEXT_WINDOW`
- Fallbacks: `MODEL_NAME="${MODEL_NAME:-sonnet}"`, `INPUT_PRICE="${INPUT_PRICE:-3.00}"`, `OUTPUT_PRICE="${OUTPUT_PRICE:-15.00}"`, `CONTEXT_WINDOW="${CONTEXT_WINDOW:-1000000}"`

**Header display:**
```bash
echo "[ token / cost ]  model: ${MODEL_NAME}  (~est.)"
echo ""
```

**Accumulator loop:**
- `TOTAL_TOKENS=0`, `EVENT_COUNT=0`
- `tail -F "${NDJSON}" 2>/dev/null | while IFS= read -r line; do`
- Per event: `line_len=${#line}`, `est_tokens=$(( line_len / 4 ))`, accumulate
- Refresh display every 5 events OR if EVENT_COUNT <= 5
- Cost calculation via inline `node -e`:
  - 70/30 input/output split: `cost = (tokens * 0.7 / 1e6 * inputPrice) + (tokens * 0.3 / 1e6 * outputPrice)`
  - Output: `$X.XXXX ~est.`
  - Fallback: `$?.???? ~est.` if node fails
- Display using ANSI cursor repositioning (`\033[3;1H\033[J`):
  ```
    Events:      {N}
    Tokens:      {N} (~est.)
    Cost:        $X.XXXX (~est.)
  ```
- Footer (dimmed gray `\033[90m`):
  ```
    Heuristic: chars/4. Model: {modelName}.
    Pricing: ${inputPrice}/${outputPrice} per MTok (input/output)
    Scope: orchestrator only (not subagents)
  ```

**Key requirements:**
- Every numeric display MUST include `(~est.)` suffix — TOKN-01 mandates this
- The pricing lookup MUST use `model-profiles.cjs` — TOKN-02 mandates per-model pricing from existing config
- The cost MUST differ between model tiers — sonnet at 100k tokens: $0.66, haiku: $0.22
- No `bc` usage — Node.js for all float math
- Node cost subprocess spawns only every 5 events, not per-event
