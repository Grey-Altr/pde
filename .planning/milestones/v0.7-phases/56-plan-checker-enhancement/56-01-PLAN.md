**Question:** "Do all upstream phases this plan depends on have completed execution?"

**Process (step-by-step):**
1. Read the current phase number from the verification context
2. Run: `node "$HOME/.claude/pde-os/engines/gsd/bin/gsd-tools.cjs" roadmap analyze --raw` to get phases array with `depends_on` and `disk_status` per phase
3. Parse the current phase's `depends_on` field (string like "Phase 54" or "Phase 55, Phase 56" or null)
4. Extract phase numbers from the depends_on string using regex `/Phase\s+(\d+)/g`
5. For each extracted phase number, look up its `disk_status` in the roadmap analyze output
6. Classification: `disk_status === "complete"` = no gap; `disk_status === "partial"` = CONCERNS gap; all others (planned, researched, discussed, empty, no_directory) = FAIL gap
7. DEPS-06 scope constraint: ONLY check direct depends_on phases — do NOT recurse to transitive dependencies
8. If any gaps found, write DEPENDENCY-GAPS.md to the phase directory
9. If no depends_on (null), write DEPENDENCY-GAPS.md with result: pass, gap_count: 0

**DEPENDENCY-GAPS.md artifact format:**
```markdown
---
phase: {phase-slug}
generated: "{ISO timestamp}"
result: {pass | concerns | fail}
gap_count: {N}
---

# Phase {N}: Dependency Gaps

**Generated:** {timestamp}
**Result:** {PASS | CONCERNS | FAIL}
**Gaps found:** {N}

## Dependency Check

| Upstream Phase | Status | Severity | Resolution Options |
|----------------|--------|----------|-------------------|
| Phase {N} | {disk_status} | {PASS/CONCERNS/FAIL} | {resolution options or "---"} |

## Resolution Details

{Only present when gaps exist. For each gap:}

### Phase {N} ({disk_status})

**Resolution options:**
- **Reorder:** Execute Phase {N} before current phase
- **Add stub:** Add stub task in current phase Wave 0 that scaffolds Phase {N} output
- **Add prerequisite:** Explicitly add Phase {N} as a prerequisite in ROADMAP.md
```

**Issue output format (for each gap):**
```yaml
issue:
  plan: null
  dimension: "cross_phase_dependencies"
  severity: "{blocker | concerns}"  # blocker for planned/empty/no_directory; concerns for partial
  description: "Phase depends on Phase {N} but its disk_status is '{status}'"
  gap:
    phase: "{N}"
    dependency_type: "direct"
    upstream_status: "{status}"
  resolution_options:
    - type: "reorder"
      description: "Execute Phase {N} before current phase"
    - type: "add_stub"
      description: "Add stub task in current phase Wave 0 that scaffolds Phase {N} output"
    - type: "add_prerequisite"
      description: "Mark current phase as depending on Phase {N} in ROADMAP.md"
```

**Severity rules:**
- `disk_status === "complete"` → no issue emitted
- `disk_status === "partial"` → severity: "concerns"
- All other statuses → severity: "blocker"
