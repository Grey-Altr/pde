Create `workflows/build.md` — the full pipeline orchestrator workflow. This is the core implementation. Follow the structural conventions from `workflows/handoff.md` (purpose block, required_reading, flags, process with numbered steps).

**Structure:**

`<purpose>`: Thin orchestrator that reads designCoverage, determines completed stages, and invokes each remaining stage in canonical order via flat Skill() calls. Adds no skill logic — all behavior lives in individual skill workflows. Supports verification gates between stages (interactive mode) and auto-continue (yolo mode). Resumes from last complete stage after interruption.

`<required_reading>`: @references/skill-style-guide.md

`<flags>`: Table with --dry-run, --quick, --verbose, --force flags. --dry-run shows pipeline status without executing. --quick/--verbose/--force forwarded to sub-skills as passthrough args.

`<process>`: The main workflow in 4 orchestrator steps:

**Step 1/4: Initialize and read configuration**
- Parse $ARGUMENTS for flags (--dry-run, --quick, --verbose, --force). Build PASSTHROUGH_ARGS string from recognized flags.
- Read `.planning/config.json` via Read tool. Parse `mode` field. Default to "yolo" if absent.
- Run `pde-tools.cjs design ensure-dirs` to initialize directory structure.
- Display: `Step 1/4: Initialized. Mode: {MODE}.`

**Step 2/4: Read pipeline state and determine pending stages**
- Run coverage-check:
  ```bash
  COV=$(node "${CLAUDE_PLUGIN_ROOT}/bin/pde-tools.cjs" design coverage-check)
  if [[ "$COV" == @file:* ]]; then COV=$(cat "${COV#@file:}"); fi
  ```
- Parse the JSON result. Extract: hasDesignSystem, hasFlows, hasWireframes, hasCritique, hasIterate (default false if absent), hasHandoff.
- Check brief completion via Glob on `.planning/design/strategy/BRF-brief-v*.md`. If any file found, BRIEF_DONE = true.
- Build the stage status table (7 stages):

  | Stage | Skill | Check | Status |
  |-------|-------|-------|--------|
  | 1/7 | /pde:brief | Glob BRF-brief-v*.md | complete/pending |
  | 2/7 | /pde:system | hasDesignSystem | complete/pending |
  | 3/7 | /pde:flows | hasFlows | complete/pending |
  | 4/7 | /pde:wireframe | hasWireframes | complete/pending |
  | 5/7 | /pde:critique | hasCritique | complete/pending |
  | 6/7 | /pde:iterate | hasIterate | complete/pending |
  | 7/7 | /pde:handoff | hasHandoff | complete/pending |

- Display the table showing completed and pending stages.
- If ALL stages complete: display "All 7 stages complete. Pipeline finished." and HALT (success).
- If --dry-run: display the table and HALT without executing any stages.
- Display: `Step 2/4: Pipeline state loaded. {N}/7 stages complete, {7-N} pending.`

**Step 3/4: Execute pending stages in order**
- For EACH stage from 1-7, in order:
  - If stage is already complete: display `Stage {N}/7: /pde:{skill} — skipped (complete)` and continue to next.
  - If stage is pending:
    - Display: `Stage {N}/7: Running /pde:{skill}...`
    - Invoke: `Skill(skill="pde:{skill}", args="${PASSTHROUGH_ARGS}")`
    - After Skill returns, display: `Stage {N}/7: /pde:{skill} — done.`
    - **Verification gate** (only between stages, not after the last one):
      - If MODE == "interactive": Use AskUserQuestion with question "Stage {N}/7 (/pde:{skill}) complete. Continue to Stage {N+1}/7 (/pde:{next_skill})?" and options ["Continue", "Stop here"]. If "Stop here": display "Pipeline paused after Stage {N}/7. Resume: /pde:build (will skip completed stages automatically)" and HALT.
      - If MODE != "interactive": display `[{MODE}] Advancing to Stage {N+1}/7...` and continue.

The 7 stages and their Skill calls, in exact order:
1. `Skill(skill="pde:brief", args="${PASSTHROUGH_ARGS}")`
2. `Skill(skill="pde:system", args="${PASSTHROUGH_ARGS}")`
3. `Skill(skill="pde:flows", args="${PASSTHROUGH_ARGS}")`
4. `Skill(skill="pde:wireframe", args="${PASSTHROUGH_ARGS}")`
5. `Skill(skill="pde:critique", args="${PASSTHROUGH_ARGS}")`
6. `Skill(skill="pde:iterate", args="${PASSTHROUGH_ARGS}")`
7. `Skill(skill="pde:handoff", args="${PASSTHROUGH_ARGS}")`

**Step 4/4: Pipeline complete**
- Display: `Step 4/4: Pipeline complete. All 7 stages finished.`
- Display summary: list all 7 stages with "done" status.
- Display: `Design artifacts are in .planning/design/. Run /pde:handoff --verbose to review the implementation spec.`

**CRITICAL anti-patterns to avoid (from RESEARCH.md):**

- NEVER assume `hasIterate` exists in coverage JSON — default to false if absent (added at runtime by /pde:iterate).
- NEVER pass mode flags to sub-skills — each skill reads config.json independently.
